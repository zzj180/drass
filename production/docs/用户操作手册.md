## 系统概述

### 系统简介

磐石数据合规分析系统是一个基于AI的数据合规管理和风险识别平台，提供：

- **数据合规分析**：智能识别数据合规风险点
- **文档管理**：支持多种格式的文档上传和管理
- **审计日志**：完整的系统操作审计追踪
- **实时监控**：系统性能和合规指标监控

### 系统架构

```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   前端界面      │    │   后端API       │    │   AI模型服务    │
│   (React)       │◄──►│   (FastAPI)     │◄──►│   (VLLM)        │
└─────────────────┘    └─────────────────┘    └─────────────────┘
         │                       │                       │
         │                       │                       │
         ▼                       ▼                       ▼
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   文档存储      │    │   向量数据库    │    │   监控系统      │
│   (文件系统)    │    │   (ChromaDB)    │    │   (Prometheus)  │
└─────────────────┘    └─────────────────┘    └─────────────────┘
```

## 核心功能使用指南

### 1. 知识库管理

#### 1.1 文档上传

**支持的文件格式：**
- PDF、DOC、DOCX
- XLS、XLSX
- PPT、PPTX
- TXT、MD、CSV

**上传步骤：**
1. 点击"知识库"菜单
2. 点击"上传文档"按钮
3. 选择文件或拖拽文件到上传区域
4. 等待上传和处理完成
5. 文档将自动进行文本提取和向量化

**批量上传：**
- 支持同时选择多个文件
- 系统会显示上传进度
- 失败的文件会显示错误信息

#### 1.2 文档管理

**文档列表功能：**
- 查看所有已上传的文档
- 按名称、类型、上传时间排序
- 搜索和过滤文档
- 删除不需要的文档

**文档详情：**
- 查看文档基本信息
- 查看处理状态
- 下载原始文件

### 2. 数据合规分析助手

#### 2.1 基础对话

**使用方法：**
1. 点击"数据合规分析助手"菜单
2. 在输入框中输入问题
3. 点击发送或按Enter键
4. 等待AI分析并返回结果

**示例问题：**
- "什么是数据合规？"
- "如何保护个人信息？"
- "GDPR的主要要求是什么？"
- "数据泄露后应该如何处理？"

#### 2.2 基于文档的分析

**RAG模式：**
- 系统会自动检索相关文档
- 基于知识库内容提供专业回答
- 回答会包含文档引用

**切换模式：**
- 在设置中可以开启/关闭RAG模式
- RAG模式：基于知识库的专业回答
- 直接模式：AI模型的自由回答

#### 2.3 会话管理

**会话功能：**
- 自动保存对话历史
- 支持多会话切换
- 可以删除历史会话
- 支持会话重命名

### 3. 审计日志

#### 3.1 日志查看

**日志类型：**

## 故障排除

### 常见问题

#### 1. 无法访问系统

**可能原因：**
- 网络连接问题
- 服务未启动
- 端口被占用
- 防火墙阻止

**解决方法：**
1. 检查网络连接
2. 确认服务状态：`./production/health-check.sh`
3. 检查端口占用：`lsof -i :5173,8888`
4. 检查防火墙设置

#### 2. 文档上传失败

**可能原因：**
- 文件格式不支持
- 文件大小超限
- 存储空间不足
- 权限问题

**解决方法：**
1. 检查文件格式和大小
2. 检查磁盘空间：`df -h`
3. 检查文件权限
4. 查看错误日志

#### 3. AI分析无响应

**可能原因：**
- VLLM服务未启动
- 模型加载失败
- 内存不足
- 网络超时

**解决方法：**
1. 检查VLLM服务：`curl -H "Authorization: Bearer 123456" http://localhost:8001/v1/models`
2. 检查系统资源：`free -h`
3. 重启AI服务
4. 调整超时设置

#### 4. 监控数据不显示

**可能原因：**
- Prometheus未启动
- 指标收集失败
- Grafana配置错误
- 网络连接问题

**解决方法：**
1. 检查Prometheus状态：`docker ps | grep prometheus`
2. 检查指标端点：`curl http://localhost:9090/metrics`
3. 重启监控服务
4. 检查Grafana数据源配置
