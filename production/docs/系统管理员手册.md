## 部署管理

### 生产环境部署

#### 1. 系统要求

**硬件要求：**
- CPU: 8核心以上
- 内存: 32GB以上
- 存储: 500GB以上SSD
- GPU: 支持CUDA的显卡（可选）

**软件要求：**
- Ubuntu 22.04 LTS
- Docker 20.10+
- Docker Compose 2.0+
- Python 3.9+
- Node.js 18+

#### 2. 部署步骤

**快速部署：**
```bash
# 1. 克隆项目
git clone <repository-url>
cd drass

# 2. 执行生产环境部署
./deployment/production/deploy-production.sh

# 3. 启动监控服务
./deployment/production/start-monitoring.sh

# 4. 验证部署
./production/health-check.sh
```

**详细部署：**
```bash
# 1. 检查系统依赖
./deployment/production/deploy-production.sh check

# 2. 创建目录结构
./deployment/production/deploy-production.sh setup

# 3. 配置环境变量
./deployment/production/deploy-production.sh config

# 4. 启动所有服务
./deployment/production/deploy-production.sh start

# 5. 验证服务状态
./deployment/production/deploy-production.sh verify
```

#### 3. 配置管理

**环境变量配置：**

## 服务管理

### 服务启动和停止

#### 1. 手动管理

**启动服务：**
```bash
# 启动VLLM服务
cd /home/qwkj/drass
python -m vllm.entrypoints.openai.api_server \
    --model /models/Qwen2.5-8B-Instruct \
    --host 0.0.0.0 --port 8001 \
    --api-key 123456 &

# 启动嵌入服务
python -m vllm.entrypoints.openai.api_server \
    --model /models/Qwen3-Embedding-8B \
    --host 0.0.0.0 --port 8010 \
    --api-key 123456 &

# 启动重排序服务
python -m vllm.entrypoints.openai.api_server \
    --model /models/Qwen3-Reranker-8B \
    --host 0.0.0.0 --port 8012 \
    --api-key 123456 &

# 启动后端API
cd services/main-app
uvicorn app.main:app --host 0.0.0.0 --port 8888 --workers 4 &

# 启动前端服务
cd frontend
npm run dev -- --host 0.0.0.0 --port 5173 &
```

**停止服务：**
```bash
# 查找并停止进程
pkill -f "vllm.entrypoints.openai.api_server"
pkill -f "uvicorn app.main:app"
pkill -f "npm run dev"

# 或使用端口停止
lsof -ti:8001 | xargs kill -9
lsof -ti:8010 | xargs kill -9
lsof -ti:8012 | xargs kill -9
lsof -ti:8888 | xargs kill -9
lsof -ti:5173 | xargs kill -9
```

#### 2. 系统服务管理

**创建systemd服务：**

#### 2. 告警配置

**告警规则：**
```yaml
# 服务可用性告警
- alert: ServiceDown
  expr: up == 0
  for: 1m
  labels:
    severity: critical
  annotations:
    summary: "服务 {{ $labels.instance }} 不可用"

# 高CPU使用率告警
- alert: HighCPUUsage
  expr: 100 - (avg by(instance) (irate(node_cpu_seconds_total{mode="idle"}[5m])) * 100) > 80
  for: 5m
  labels:
    severity: warning
  annotations:
    summary: "CPU使用率过高"
```

**告警通知：**
- 邮件通知
- 短信通知
- 钉钉/企业微信通知
- Webhook通知

## 故障处理

### 常见故障

#### 1. 服务无法启动

**诊断步骤：**
```bash
# 1. 检查端口占用
lsof -i :8001,8010,8012,8888,5173

# 2. 检查进程状态
ps aux | grep -E "(vllm|uvicorn|npm)"

# 3. 查看错误日志
tail -f /home/qwkj/drass/production/logs/app/*.log

# 4. 检查系统资源
free -h
df -h
```

**解决方案：**
- 释放被占用的端口
- 重启相关服务
- 清理系统资源
- 检查配置文件

#### 2. 性能问题

**性能诊断：**
```bash
# 1. 检查系统负载
uptime
top
htop

# 2. 检查内存使用
free -h
cat /proc/meminfo

# 3. 检查磁盘I/O
iostat -x 1
iotop

# 4. 检查网络连接
netstat -an | grep ESTABLISHED | wc -l
```

**性能优化：**
- 调整工作进程数
- 优化内存分配
- 使用缓存机制
- 升级硬件配置

#### 3. 数据问题

**数据诊断：**
```bash
# 1. 检查数据库完整性
sqlite3 /home/qwkj/drass/data/database.db "PRAGMA integrity_check;"

# 2. 检查文件权限
ls -la /home/qwkj/drass/data/

# 3. 检查磁盘空间
```
