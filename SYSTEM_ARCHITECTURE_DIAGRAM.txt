启动流程 (Startup Flow)
=====================

1. 环境检查 (Environment Check)
   ├── 检查已运行服务
   ├── 创建必要目录
   └── 配置环境变量

2. AI服务启动 (AI Services Startup)
   ├── vLLM LLM服务 (DeepSeek-R1模型)
   ├── vLLM Embedding服务 (Qwen3-Embedding模型)
   └── vLLM Reranking服务 (Qwen3-Reranker模型)

3. 数据存储启动 (Data Storage Startup)
   ├── PostgreSQL数据库
   ├── Redis缓存
   └── ChromaDB向量数据库

4. 后端API启动 (Backend API Startup)
   ├── FastAPI应用
   ├── 中间件配置
   └── 路由注册

5. 前端服务启动 (Frontend Startup)
   ├── React开发服务器
   ├── Vite构建工具
   └── 热重载配置

6. 健康检查 (Health Check)
   ├── 服务状态验证
   ├── 端口监听检查
   └── 功能测试

服务依赖关系 (Service Dependencies)
==================================

前端 (Frontend) ──→ 后端API (Backend API)
    │                      │
    │                      ├──→ vLLM LLM服务
    │                      ├──→ vLLM Embedding服务
    │                      ├──→ vLLM Reranking服务
    │                      ├──→ ChromaDB向量存储
    │                      ├──→ PostgreSQL数据库
    │                      └──→ Redis缓存

数据流向 (Data Flow)
==================

用户输入 ──→ 前端界面 ──→ 后端API ──→ AI服务处理 ──→ 数据存储 ──→ 响应返回

1. 用户在前端输入查询
2. 前端发送HTTP请求到后端API
3. 后端API调用AI服务进行文本处理
4. AI服务查询向量数据库获取相关信息
5. 结合用户数据和AI模型生成响应
6. 响应通过API返回给前端显示

故障排除 (Troubleshooting)
=========================

常见问题及解决方案:

1. 端口冲突
   ├── 检查: lsof -i :端口号
   ├── 解决: kill -9 PID
   └── 预防: 启动前检查端口

2. GPU内存不足
   ├── 检查: nvidia-smi 或 rocm-smi
   ├── 解决: 调整gpu_memory_utilization参数
   └── 预防: 监控GPU使用率

3. 服务启动失败
   ├── 检查: 查看对应日志文件
   ├── 解决: 检查依赖和配置
   └── 预防: 定期更新依赖

4. 数据库连接失败
   ├── 检查: 数据库服务状态
   ├── 解决: 重启数据库服务
   └── 预防: 配置自动重启
